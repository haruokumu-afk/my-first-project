# SEO記事作成ワークフロー（完全版）

## 全体フロー
1. ジャンル確認・トピック選定
2. HPコラム一覧取得・既存記事との重複チェック
3. 競合リサーチ・キーワード選定
4. Deep Research（情報収集）
5. 記事本文作成（過去記事フォーマット準拠）
6. **SEOチェック** ← NEW
7. **ファクトチェック（ハルシネーション検証）** ← 強化版
8. タイトル・メタディスクリプション作成
9. タイトルに合わせて冒頭文を微調整
10. HTML化（目次含む・過去記事テンプレート準拠）
11. 画像プロンプト・ファイル名・alt・挿入位置作成（3枚）
12. 完成サマリー出力

---

## STEP 1：ジャンル確認・トピック選定

### 1-1. ターゲット読者の確認
記事作成前に以下を確認する：
- 想定読者層（例：離婚を検討中の30〜50代女性、老後の住まいを検討中のシニア層 等）
- 読者の悩み・検索動機（何を知りたくてGoogle検索するか）
- 集客導線（記事からどのページ・サービスに誘導するか）

### 1-2. トピック選定の基準
以下の条件を満たすトピックを優先する：
- 想定読者の悩みに直結している
- 既存記事と重複しない（STEP 2で確認）
- 検索需要がある（関連キーワードの検索ボリュームが見込める）
- 競合記事の質が低く、差別化できる余地がある

### 1-3. 確認・承認
トピック案を提示し、ユーザーの承認を得てからSTEP 2へ進む。
勝手にトピックを確定して先に進まないこと。

---

## STEP 2：HPコラム一覧取得・既存記事との重複チェック

### 2-1. 既存記事一覧の取得
- ユーザーからHPのコラムURL一覧、またはサイトマップを取得する
- 取得できない場合はユーザーに確認する（勝手にWeb検索で代替しない）

### 2-2. 重複チェックの基準
以下のいずれかに該当する場合は「重複あり」と判断し、別トピックを提案する：
- まったく同じキーワード・テーマを扱っている既存記事がある
- 既存記事と70%以上の内容が重複すると予想される

### 2-3. 類似記事がある場合の対応
- 既存記事と「切り口・読者層・検索意図」が明確に異なれば、差別化ポイントを明示した上で作成可
- 差別化が難しい場合は既存記事のリライト提案に切り替える

---

## STEP 3：競合リサーチ・キーワード選定

### 3-1. メインキーワードの選定
- 記事テーマを最もシンプルに表す2〜4語のキーワードを1つ選ぶ
- 例：「離婚 住まい 女性」「老後 一人暮らし マンション」

### 3-2. サブキーワード・共起語の収集
Web検索（Google検索）で以下を確認する：
- 「関連する検索キーワード」「他の人はこちらも検索」欄のワード
- 検索結果上位5記事の見出し（h2・h3）構成
- サジェストキーワード（検索窓に入力したときに出てくる候補）

### 3-3. キーワードマップの整理
```
メインキーワード：〇〇
サブキーワード：〇〇、〇〇、〇〇
共起語（記事内に自然に含めたいワード）：〇〇、〇〇、〇〇
競合上位記事でカバーされているトピック：
　・〇〇
　・〇〇
差別化ポイント（自記事の独自切り口）：〇〇
```

### 3-4. 確認・承認
キーワードマップをユーザーに提示し、承認を得てからSTEP 4へ進む。

## STEP 4：Deep Research（情報収集）

### 4-1. 情報収集の優先ソース
以下の順で信頼性の高い情報を収集する：
1. 政府機関・公的団体（国土交通省・厚生労働省・総務省・法務省 等）
2. 公益財団法人・業界団体（不動産流通機構・住宅金融支援機構 等）
3. 大手金融機関・不動産会社の公式サイト
4. 査読済み学術論文・調査レポート

個人ブログ・まとめサイト・SEOアフィリエイト記事は参考にしない。

### 4-2. 収集する情報の種類
- 統計データ・調査結果（数値・割合・推移）
- 法律・税制・制度の要件・条件・期限
- 用語の定義・正式名称
- 専門家の見解（引用元を明確にする）

### 4-3. 情報整理フォーマット
収集した情報を以下の形式で整理してからSTEP 5へ進む：

```
【収集情報まとめ】
章構成案：
　第1章：〇〇
　第2章：〇〇
　…

使用予定のデータ・根拠：
　・〇〇（出典：〇〇、URL：https://...）
　・〇〇（出典：〇〇、URL：https://...）

不明点・要確認事項：
　・〇〇
```

## STEP 5：記事本文作成（過去記事フォーマット準拠）

### 5-0. 【最重要】記事の根本姿勢

**この記事を書く相手は「情報を知りたい人」ではなく「悩みを抱えた人」である。**

ターゲット読者は今、不安・迷い・焦り・孤独感の中でこの記事にたどり着いている。
正確な情報を届けることは当然の前提として、その上で「この記事を書いた人は私のことをわかってくれている」と感じてもらうことが最優先ゴールである。

法律の条文を写したような文章、箇条書きを並べただけの解説、感情のない事実の羅列は**禁止**。
読者が「続きを読みたい」と思い、最後まで読み終えて「相談してみようかな」と思える文章を書く。

---

### 5-1. 文体・トーンの原則

**基本姿勢：「専門家が隣に座って話しかける」イメージ**

- 丁寧語（です・ます調）を基本とする
- 読者を「あなた」と呼びかけてよい。距離感を縮める
- 専門用語は初出時に必ずカッコで補足説明する（例：「譲渡所得税（不動産を売却した際にかかる税金）」）
- 断定的な表現は避け、条件・前提を添える（「〜の場合があります」「〜傾向があります」）

**やってはいけない文体（Bad examples）：**
```
❌ 「不動産売却に係る譲渡所得税の計算式は以下のとおりである。」
❌ 「以下の要件を満たす場合、控除の適用を受けることができる。」
❌ 「メリットは以下の3点が挙げられる。①〜②〜③〜」
```

**目指す文体（Good examples）：**
```
✅ 「不動産を売ったときに気になるのが税金ですよね。実は、条件によっては
   大幅に節税できる制度があります。順番に確認していきましょう。」
✅ 「『自分は使えるの？』と思った方のために、具体的な条件をわかりやすく
   整理しました。」
✅ 「3つのメリットがありますが、特に知っておいてほしいのが1つ目です。」
```

---

### 5-2. 読者の感情を動かす文章テクニック

#### A. 共感から入る（章の書き出しルール）
各h2章の書き出しは、読者の心境・状況への共感または問いかけから始める。
いきなり「〇〇とは〜」「〇〇の方法は〜」と始めない。

```
✅ 「離婚後の住まい探しで、多くの女性が直面するのが『収入が一本になった状態で
   賃貸を借りられるのか』という不安です。」
✅ 「『老後はどこに住めばいいのか』——この問いに、明確な答えを出せている人は
   意外と少ないのではないでしょうか。」
```

#### B. 読者の代弁をする
読者が心の中で思っていそうなことを、先回りして言葉にする。
「〜と感じる方もいるかもしれません」「〜と思った方もいるでしょう」という橋渡し表現を章中に1〜2回入れる。

#### C. 具体的なシーンで想像させる
抽象的な説明だけで終わらず、「こういうケースの場合」という具体的な場面・人物像を使って説明する。
数字や条件も、「たとえば〜の場合は〜円になります」と計算例で示す。

#### D. 「だから何？」を先に答える
情報を説明する前に、「これを知ると何が得か」「なぜこれが重要か」を一文で先出しする。
読者に「この先を読む理由」を与え続けることで離脱を防ぐ。

```
✅ 「この制度を知っているかどうかで、手取りが数十万円変わることがあります。」（→詳細説明）
✅ 「実はここが、多くの人が見落としているポイントです。」（→詳細説明）
```

#### E. 章末に「次へのつなぎ」を入れる
各h2章の最後の1〜2文で、次の章への期待感を高める一言を添える。
読者が「次の章も読もう」と思う導線をつくる。

```
✅ 「では、具体的にどんな物件を選べばよいのでしょうか。次の章で詳しく見ていきます。」
✅ 「費用のことが気になり始めた方も多いと思います。次章でまとめて解説します。」
```

---

### 5-3. 構成ルール

- **h2は5〜6個**に絞る（厳守）
- 各h2の下にh3を2〜4個ぶら下げる論理的な階層構造にする
- 各章の文字数目安：400〜600文字（h3含む合計）
- 冒頭文（リード文）：150〜200文字。「誰に向けた記事か」「読むと何がわかるか」を明示する

**章の役割設計：**
| 位置 | 役割 | トーン |
|------|------|--------|
| 第1章 | 読者の悩み・現状への共感。「あなたのことをわかっている」と伝える | 共感・寄り添い |
| 第2〜4章 | 具体的な情報・方法・比較。記事の核心 | 丁寧な解説・背中を押す |
| 第5章 | 注意点・落とし穴。読者を守る視点 | 親身な警告 |
| 最終章 | まとめ＋次のアクション（CTA）。前向きな締め | 背中を押す・希望 |

---

### 5-4. 冒頭文テンプレート

冒頭文は「読者の悩みを言語化する」→「この記事で解決できると宣言する」の2段構成にする。

```
〇〇を考えはじめたとき、「〇〇できるのだろうか」「〇〇はどうすればいいのか」
と不安になる方は少なくありません。

この記事では、〇〇について〔具体的なポイント1〕〔ポイント2〕〔ポイント3〕
の観点からわかりやすく解説します。
〇〇で悩んでいる方の、少しでも力になれれば幸いです。
```

**禁止パターン：**
```
❌ 「〇〇について解説します。」（素っ気なく、読む動機が生まれない）
❌ 「〇〇とは何でしょうか。」（教科書の書き出しになっている）
```

---

### 5-5. まとめ章のテンプレート

まとめは「ポイントの箇条書き」→「読者への言葉がけ」→「CTA」の3段構成にする。

```
## まとめ：〇〇のために、今できること

ここまで〇〇についてお伝えしてきました。改めてポイントを整理します。

・〇〇
・〇〇
・〇〇

〇〇は、情報が多く「自分の場合はどうなのか」が見えにくいテーマです。
一人で抱え込まず、まずは専門家に話を聞いてみることをおすすめします。

（内部リンク・CTAをここに配置）
```

---

### 5-6. 文字数チェック（必須）
本文完成後、共通ルールのPythonスクリプトで必ず文字数を実測すること。
4,500〜5,000文字の範囲に収まるまで調整を繰り返す。

## STEP 6：SEOチェック（記事本文完成後、ファクトチェック前に実施）

記事本文が完成したら、以下の観点でSEO品質を検証する。

### 6-1. 検索意図との整合性
- ターゲットキーワードで実際にGoogle検索し、上位5記事の構成（見出し・章立て）を取得する
- 上位記事がカバーしていて、自記事に欠けているトピックがないか確認する
- 欠けている重要トピックがあれば記事に追加する
- 逆に、上位記事にない独自の切り口（差別化ポイント）が含まれているか確認する

### 6-2. キーワード配置チェック
以下の箇所にターゲットキーワード（またはその自然な変形）が含まれているか確認する：
- タイトル（h1）
- 最初の見出し（h2）
- 冒頭文（最初の100文字以内）
- 各h2見出しの少なくとも半数
- メタディスクリプション
- 画像のalt属性
※ 不自然なキーワード詰め込みは逆効果。読んで違和感がないことを最優先する。

### 6-3. 共起語・関連語の網羅度
- ターゲットキーワードの関連検索ワード（Google検索の「他の人はこちらも検索」等）をWeb検索で取得する
- 重要な共起語が記事内に自然に含まれているかチェックする
- 不足している場合は、文脈に沿う形で本文に組み込む

### 6-4. 構成・可読性チェック
- h2は4〜8個の範囲に収まっているか
- h2の下にh3がぶら下がる論理的な階層構造になっているか
- 1つのh2セクションが長すぎないか（目安：600文字以内で1つのh3またはポイントを区切る）
- 冒頭文で「この記事を読むと何がわかるか」が明確に伝わるか
- まとめ（最終章）にCTA（相談誘導・問い合わせリンク等）が含まれているか

### 6-5. 内部リンク・導線設計
- 過去のコラム記事で関連性の高いものを2〜3本ピックアップし、本文中に自然な形で内部リンクを設置する
- 集客導線ページ（例：woman_lp 等）へのリンクをまとめセクションに含める

### 6-6. SEOチェック結果の出力
以下のフォーマットで結果を報告し、必要な修正を実行する：

```
【SEOチェック結果】
■ 検索意図との整合性：OK / 要追加トピックあり（詳細：〜）
■ キーワード配置：OK / 不足箇所あり（詳細：〜）
■ 共起語網羅度：OK / 追加推奨ワードあり（詳細：〜）
■ 構成・可読性：OK / 要改善（詳細：〜）
■ 内部リンク：設置済み○本 / 追加推奨（詳細：〜）
■ 修正実施：あり / なし
```

修正を実施した場合は、修正箇所を明記すること。

## STEP 7：ファクトチェック（ハルシネーション検証）強化版

SEOチェック・修正が完了した記事に対して、以下の手順でファクトチェックを行う。

### 7-1. 事実主張の全抽出
記事本文を精読し、以下に該当する記述を**一つ残らず**箇条書きで抽出する。
見落とし厳禁。章ごとに順番に抽出すること。

対象となる記述の種類：
- 数値（金額・年齢・期間・割合・面積・統計データ等）
- 法律・税制・制度に関する記述（要件・期限・条件・届出先等）
- 定性的な主張（「〜しやすい」「〜が多い」「〜傾向がある」「〜と言われている」等）
- 比較・優劣の表現（「〜より安い」「〜が一般的」「〜のほうが有利」等）
- 固有名詞に紐づく情報（UR賃貸の条件、フラット35の要件、特定の制度名等）
- 因果関係の主張（「〜なので〜になる」「〜すると〜できる」等）

### 7-2. 個別検証
抽出した事実主張をすべてWeb検索で裏取りする。
以下のフォーマットで検証結果を**一覧表として**報告すること。

| # | 章 | 記事内の記述 | 検証結果 | ソースURL | 判定 |
|---|---|-----------|--------|----------|------|
| 1 | 第1章 | 女性の平均寿命は87.14歳 | 厚労省令和5年簡易生命表で確認 | https://... | ✅ |
| 2 | 第2章 | コンパクトマンションは資産価値が落ちにくい | 立地・築年数による。一概には言えない | https://... | ⚠️ |

判定の基準：
- ✅ 確認済み：信頼できるソース（政府機関・公的団体・大手金融機関等）で裏付けあり
- ⚠️ 要修正：不正確・誇張・条件や前提の記載不足・出典が曖昧
- ❌ 削除推奨：裏付けなし・明確な誤り・誤解を招く表現

### 7-3. 修正ルール
⚠️・❌の項目について記事本文を修正する。修正時の原則：
- 「〜と言われています」等の曖昧な出典表現 → 具体的なソース名に置き換え（例：「厚生労働省の調査によると」）
- 裏付けが取れない具体的数値 → 削除するか「※金額は個別の条件により異なります」等の注釈に置き換え
- 定性的な主張 → 条件・前提を明記する（例：「立地や築年数にもよりますが、〜の傾向があります」）
- 比較表現 → 比較の根拠を明示する
- 法律・税制の記述 → 施行年・改正年を明記し、「最新の情報は専門家にご確認ください」を添える

### 7-4. 再チェック（ループ）【省略厳禁】
修正を行った場合、このステップは絶対にスキップしてはならない。
修正後の記事に対してSTEP 7-1〜7-3を再度実行する。

手順：
1. 修正後の記事全文を再度精読し、事実主張を全抽出する
2. 前回⚠️・❌だった項目の修正後の表現を重点的に検証する
3. 新たに追加された記述（SEOチェックで追加した内容等）も検証対象に含める
4. 再チェック結果を一覧表で報告し、「再チェック完了：⚠️残り○件、❌残り○件」と明記する

ループ条件：
- ⚠️・❌がゼロになるまで繰り返す（最大3ループ）
- 3ループで解消しない項目には【要専門家確認】タグを付けて残す
- 再チェックを実行したことをfact-check-report.mdに記録すること（「再チェック1回目：実施済み、結果：全件✅」等）

※ 今回の記事で再チェックをスキップした結果、寡婦控除・ひとり親控除の要件誤りという重大な問題を見逃しかけた。この教訓を忘れないこと。

### 7-5. 不動産領域の特別チェック項目
不動産SEO記事では以下の項目に特に注意し、必ず個別に検証すること：
- 税金の計算例・課税タイミング（譲渡所得税・3,000万円特別控除の適用条件等）
- 年金分割の計算条件（合意分割と3号分割の違い・対象期間等）
- ローン審査基準の具体的数値（年齢制限・収入要件・返済比率等）
- 法改正の影響を受ける記述（宅建業法・民法改正等の施行日）
- 費用の相場感（仲介手数料率・登記費用・引越し費用等）

### 7-6. ファクトチェックレポート出力
全工程完了後、以下を別ファイル（fact-check-report.md）として出力する：

1. 検証結果の全一覧表（最終版）
2. 修正履歴（初稿の記述 → 修正後の記述）
3. 【要専門家確認】タグ付き項目の一覧
4. 宅建士・税理士・FPへの確認推奨項目リスト
5. 使用したソースURL一覧

このレポートは記事納品時に必ずセットで出力すること。
人間の最終レビュー（特に法律・税制・制度関連）はこのレポートをもとに行う。

## STEP 8：タイトル・メタディスクリプション作成

### 8-1. タイトル（h1）の作成基準
- **文字数**：28〜35文字（全角）を目安とする
- **必須要素**：メインキーワードを自然な形で含める
- **形式**：読者の悩みや疑問を反映した訴求型が望ましい
  - 例：「〇〇する方法」「〇〇のポイントを解説」「〇〇を徹底比較」「〇〇前に知っておきたいこと」
- **禁止**：誇大表現（「完全ガイド」「絶対」「必ず」等）の過剰使用

**タイトル候補の出し方**：3〜5案提示してユーザーに選んでもらう。

### 8-2. メタディスクリプションの作成基準
- **文字数**：110〜130文字（全角）を目安とする
- **必須要素**：
  - メインキーワードを含む
  - 記事で得られる情報・解決できる悩みを明記
  - 読者に読む動機を与えるCTA的な一文で締める
- **形式例**：
  ```
  〇〇を検討中の方へ。この記事では〇〇・〇〇・〇〇について詳しく解説します。〇〇でお悩みの方はぜひ参考にしてください。
  ```

### 8-3. 出力フォーマット
```
【タイトル候補】
案1：〇〇（〇〇文字）
案2：〇〇（〇〇文字）
案3：〇〇（〇〇文字）

【メタディスクリプション】
〇〇（〇〇文字）
```

## STEP 9：タイトルに合わせて冒頭文を微調整

### 9-1. 調整の目的
STEP 8で確定したタイトルと冒頭文の表現・訴求軸を揃える。
タイトルと冒頭文が別の方向を向いていると、読者の離脱率が上がる。

### 9-2. 確認ポイント
- タイトルに含まれるキーワードが冒頭100文字以内に自然に登場しているか
- タイトルが約束した内容（「〇〇の方法」等）を冒頭文で補強しているか
- 読者の悩みへの共感 → この記事で解決できる宣言、の流れになっているか

### 9-3. 調整が必要なケース
- タイトルを変更したことで冒頭のキーワードがずれた場合
- タイトルの訴求軸（比較・方法・注意点 等）と冒頭文のトーンが合っていない場合

修正箇所は最小限にとどめ、冒頭文全体を書き直さない。

## STEP 10：HTML化（目次含む・過去記事テンプレート準拠）

過去記事のHTML構造・文体・トーンに準拠してHTML化する。
以下の表記ルールを必ず守ること。

### 10-1. 補足・ワンポイント情報の表記ルール

1. 【】で囲まれた見出し（例：【ワンポイント：〇〇】）の直後に本文が続く場合、
   見出しと本文の間に `<p>&nbsp;</p>` を挿入する
2. 補足ブロック内で「・」や「●」で始まるリスト項目がある場合、
   リストの前後にも `<p>&nbsp;</p>` を挿入して視覚的に区切る
3. リスト項目同士の間にも `<p>&nbsp;</p>` を挿入して1項目ずつ読みやすくする
4. 補足ブロックの最後に注意書き（「適用条件は〜」等）がある場合、
   リストとの間に `<p>&nbsp;</p>` を挿入する

**記述例：**
```html
<p><span style="font-size:16px;"><b>【ワンポイント：税制優遇も忘れずに】</b></span></p>

<p>&nbsp;</p>

<p><span style="font-size:16px;">合計所得金額が500万円以下の場合、以下の税制優遇が受けられる可能性があります。</span></p>

<p>&nbsp;</p>

<p><span style="font-size:16px;">・寡婦控除（27万円）：離婚後に再婚しておらず、同居の親など生計を一にする扶養親族がいる場合が対象です。</span></p>

<p>&nbsp;</p>

<p><span style="font-size:16px;">・ひとり親控除（35万円）：離婚後に再婚しておらず、生計を一にする子を扶養している場合が対象です。</span></p>

<p>&nbsp;</p>

<p><span style="font-size:16px;">適用条件は個人の状況によって異なります。年末調整や確定申告での申告が必要ですので、離婚後は税務署や税理士にご確認ください。</span></p>
```

### 10-2. 内部リンクの表記ルール

1. 内部リンクは文字色を青（`color:#0000FF`）にする
2. リンクは必ず新しいタブで開くよう `target="_blank"` を付ける
3. `rel="noopener"` を合わせて付けてセキュリティ上の問題を防ぐ

**記述例：**
```html
<a href="https://example.com/column/xxx" target="_blank" rel="noopener" style="color:#0000FF;">関連記事のタイトル</a>
```

---

### 10-3. メリット・デメリットの表記ルール

1. メリット・デメリットの各項目の先頭に「●」を付ける
2. 本文説明（リード文）とメリット・デメリットリストの間に `<p>&nbsp;</p>` を挿入する
3. メリット・デメリット各項目の間にも `<p>&nbsp;</p>` を挿入して1項目ずつ読みやすくする

**記述例：**
```html
<p><br />
<span style="font-size:16px;">60歳以上を対象とした、安否確認・生活相談サービス付きの賃貸住宅。</span></p>

<p>&nbsp;</p>

<p><span style="font-size:16px;">●<b>メリット：</b>年齢を理由に断られない、安否確認などの見守りサービスがある、自由度が高い（外出・自炊も可）</span></p>

<p>&nbsp;</p>

<p><span style="font-size:16px;">●<b>デメリット：</b>一般賃貸より月額費用が高め（目安：月5〜25万円程度）、要介護度が重くなると退去を求められる場合もある</span></p>
```

## 全工程共通ルール
- 記事の対象読者と集客導線を常に意識すること
- 過去記事テンプレートのHTML構造・文体・トーンを逸脱しないこと
- 各ステップの完了時に、次に何をするかを明示してから進むこと
- 判断に迷う場合は勝手に進めず、確認を求めること
- **記事本文の文字数は4,500〜5,000文字を厳守すること**（HTML化前の本文テキストベースで計測）。h2は5〜6個に絞り、各章は400〜600文字を目安にする。

### 文字数計測ルール（必須）

**STEP 5（記事本文作成）・STEP 10（HTML化）の完了時に必ず以下のPythonスクリプトで実測すること。感覚・目視での判断は禁止。**

```python
import re
with open('article_xxx.html', 'r') as f:
    content = f.read()
text = re.sub(r'<!--.*?-->', '', content, flags=re.DOTALL)
text = re.sub(r'<[^>]+>', '', text)
text = text.replace('&nbsp;', '')
text = re.sub(r'\s+', '', text)
print(f'文字数: {len(text)}文字')
```

- 計測結果が4,500文字未満 → 各章を増量して再計測
- 計測結果が5,000文字超 → 各章を削ぎ落として再計測
- 計測結果を「文字数確認：〇〇文字」として作業ログに必ず明記すること
